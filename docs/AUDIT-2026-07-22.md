# Audit report — 2026-07-22

Three independent audit passes (correctness of the conversion core; completeness against the documented contract; robustness/performance) executed by codex agents on isolated worktrees at commit `cb977d9`, with every finding verified by scratch tests, traced arithmetic, or measurements before being accepted. One panic was independently re-reproduced during synthesis. Findings are tracked as the Milestone 8 backlog in `docs/PLAN.md`.

## A. Correctness — confirmed defects

| # | Sev | Defect | Failure evidence |
|---|---|---|---|
| A1 | High | INSERT insertion points are transformed as WCS, but DXF defines group 10 of INSERT as OCS. Wrong placement for any insert with a non-default normal; preserved anchors share the bug. | Insert normal (0,1,0), point (10,20,30), block point at origin → emitted at (10,20,30); correct arbitrary-axis result is (−10,30,20). All existing tests use the default +Z normal, which is why they pass. |
| A2 | High | `--source-units` scales drawing units to meters before PROJ, which corrupts output when the source CRS is not meter-based (e.g. EPSG:2263 in US survey feet). The documented meter-CRS assumption warns but still delivers wrong coordinates. | EPSG:2263 (987000, 212000) usft should map near (−73.9901, 40.7486); the pre-scaling yields (−76.4510, 40.3178). |
| A3 | High | Zero or sub-epsilon ARC/ELLIPSE sweeps become full revolutions (`rem_euclid` + absolute epsilon promotes 0 to 2π). | ARC with start = end = 0.75 rad tessellates as a near-full circle instead of being skipped as zero-length. |
| A4 | High | A zero OCS normal silently collapses geometry to (0,0) instead of failing the entity. | Circle r=5 at (10,20,3) with normal (0,0,0) is "converted" with every coordinate (0,0). |
| A5 | Med | Collinear (zero-area) HATCH loops are accepted as rings without any diagnostic; `check_rings` treats zero-area holes as validly non-CCW. | Outer square + collinear "hole" (2,2)-(3,2)-(4,2) → Polygon with a zero-area hole, no warning, no `hatch_loops_dropped`. |
| A6 | Med | De Boor knot-denominator test uses absolute `f64::EPSILON`, so mathematically equivalent splines with tiny knot domains evaluate wrongly. | Quadratic with knots [0,0,0,1,1,1] gives midpoint (1,0.5); knots scaled to 1e-20 give (0,0). |
| A7 | Med | Boundary containment tests vertices only; segments crossing concavities/holes are misclassified. | U-shaped boundary: segment with both endpoints inside but crossing the notch reports "inside" instead of "partial". |
| A8 | Med | Null-handle INSERTs all get feature-id prefix `model-0`: `process_insert` discards the caller's document index. | Two top-level null-handle INSERTs at indices 0 and 9 collide on `model-0`. |
| A9 | Low | Unresolved model-space handles are excluded from the accounting denominator, so `unaccounted` stays 0 while a source slot failed. | One unresolved handle → `model_space_entities = 0`, one `UNRESOLVED` failure, invariant "passes". |
| A10 | Low | Zero-MAD clusters make the outlier threshold collapse to 1e-9, flagging millimeter-scale neighbors as outliers. | 20 identical UTM points + one 1 mm away → the nearby point is flagged. |

Areas attacked without finding defects: bulge sign/closing-segment conventions and sagitta tolerance (property tests), clockwise hatch arc edges across the 2π wrap, hatch even-odd nesting and repair warnings, affine composition and default-normal INSERT chains, MINSERT ids and recursion detection, BYLAYER/BYBLOCK and layer-0 inheritance, text rotation composition, `calibrate.rs`, `units.rs`, WGS84 range checks, meter-based UTM reprojection.
